Hash Table

해시 테이블(hash table), 해시 맵(hash map), 해시 표는 컴퓨팅에서 키를 값에 매핑할 수 있는 구조인, 연관 배열 추가에 사용되는 자료 구조이다. 해시 테이블은 해시 함수를 사용하여 색인(index)을 버킷(bucket)이나 슬롯(slot)의 배열로 계산한다.

시각화

선형 탐색

선형 탐색법은 새 원소 추가시 해시 값이 이미 점유된 상태라면 해시 값 + 1을 찾아보고, 해시 값 + 1이 점유되었다면 해시 값 + 2를 찾아보는 식으로 충돌을 회피한다.

  • 설명

    이름/키 해시 값 값
    정준하 6 정준하@gmail.com
    유재석 14 유재석@gmail.com
    박명수 28 박명수@gmail.com

    1. 표와 동일한 데이터가 이미 존재합니다. 우리는 "류한경(hash value 28)"을 추가한다고 가정합니다.

    2. 이는 이미 존재하는 "박명수(hash value 28)"와 해시값이 동일하기에 충돌이 발생합니다.

    3. 선형 탐색은 이미 점유된 상태라면 해시 값 + 1 을 검색하며 충돌을 회피합니다.

    4. 다음 슬라이드로 이동하세요.

  • 설명

    이름/키 해시 값 값
    정준하 6 정준하@gmail.com
    유재석 14 유재석@gmail.com
    박명수 28 박명수@gmail.com
    류한경 29 류한경@gmail.com

    1. "류한경(hash value 28)" 삽입시 "박명수(hash value 28)"와 충돌이 발생합니다.

    2. "박명수(hash value 28)"의 해시 값 + 1(29)가 점유된 상태인지 아닌지 확인합니다.

    3. 점유된 상태가 아니라면, "류한경"을 해당 위치에 추가합니다.

체이닝

체이닝은 테이블의 인덱스별로 연결리스트를 생성하여 그 안에 원소를 저장하는 기법이다. 충돌을 해결하는 가장 단순한 기법이다.
